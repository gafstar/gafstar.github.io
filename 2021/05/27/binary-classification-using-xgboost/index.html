1. Introduction ¶

The purpose of this post it to understand how to apply XGBoost to a binary classification problem. In this post we are going to see how to apply XGBoost classifier algorithm to an adult data set downloaded from UCI Machine Learning Repository.

XGBoost is an optimized gradient boosting open source library knows for its flexibility and portability. It is also a popular algorithm for machine learning competitions for structured or tabular dataset. In Kaggle competitions, it is the number one choice for tabular dataset.

2. Data Preparation¶

We are going to prepare the data for modeling. This is an important step before we create our XGBoost model. XGBoost models don't handle categorical features, so we will use one-hot-encoding to use to convert the string values to encoded integer values. We are going to address how to handle missing values. Also we will carry out some feature transformation for a few columns with many categories. This are important steps that will give our model better predictive power.

2.1 Data Cleaning - Missing Values¶

However as we shall see later in the post, there is actually a missing values that is represented as ?. First we will assume there are no missing values in the dataset and proceed to see how good the xgboost model prediction be. Later we will remove the missing value and also check if it affects our model performance. 

print (f'Rows and columns before nan values removal {df.shape}')
df['native-country'] = df['native-country'].replace('?',np.nan)
df['workclass'] = df['workclass'].replace('?',np.nan)
df['occupation'] = df['occupation'].replace('?',np.nan)

perc = df.isnull().sum().sum() * 100/df.shape[0]
df.dropna(how='any',inplace=True)

2.2 Feature Transformation¶

Next step, we are going to transform a few of the features. In this post we will only transform three categorical columns, namely, native-country, marital-status and workclass. We can also make do feature transformation for the numerical variables, but in this post we are applying only to three columns.

The purpose of this transormation step is to lower the number of level's to 2 or 3. For example, marital-status has 7 levels, however, we transform it to 2 levels, as Couples or Single. Therefore, when it is encoded the marital-status feature will only transform to 2 columns. Basically, we are lowering the complexity of the model by using less features. 

###  Categorizing countries to US and Non-US
df.loc[df['native-country'] != 'United-States', 'native-country'] = 'Non-US'
df.loc[df['native-country'] == 'United-States', 'native-country'] = 'US'

# Converting native country to integer mapping
df['native-country'] = df['native-country'].map({'US':1,'Non-US':0}).astype(int)

# Categorize marital-status into couple and single
df['marital-status'] = df['marital-status'].replace(['Divorced','Married-spouse-absent', \

2.4 Encoding Categorical Features¶

Let's select the categorical and numerical features into cat_ix and num_ix list. This list will help us separate the categorical features from the numerical features of the dataframe. In this example, we are going to use an extreme gradient boosted tree xgboost model, a powerful variation of tree-based models. Normally, when we are working with tree-based models like decision tree, random forest we don't normalize the numerical features. However, if we are working with linear models likes logistic regression we should normalize our data. 

# select categorical and numerical features
cat_ix = X.select_dtypes(include=['object', 'bool']).columns
num_ix = X.select_dtypes(include=['int64', 'float64']).columns

Next, we are going to encode the categorical features, since xgboost doesn't handle categorical features. In this example, we are going to use one-hot encoding. We are going to add the encoded features to our dataframe X, which contains the predictor features. Then we are going to remove the unencoded categorical features from the dataframe. 

categories = cat_ix
print("Starting DF shape: %d, %d" % X.shape)


for col in categories:
    s = df[col].unique()

    # Create a One Hot Dataframe with 1 row for each unique value
    one_hot_df = pd.get_dummies(s, prefix='%s_' % col)
    one_hot_df[col] = s

    print("Adding One Hot values for %s (the column has %d unique values)" % (col, len(s)))
    pre_len = len(X)

    # Merge the one hot columns
    X = X.merge(one_hot_df, on=[col], how="left")
    assert len(X) == pre_len
    print(X.shape)

As you can see the number of features has exploded from 14 variables to 116 variables, when we applied the one-hot-encoding to the categorical features. This is bound to happen as some features has many categories. For instance, if you check the education feature, it contains 16 categories by itself. Similarly, other features have similar many categories. Later we will use feature engineering to lower the number of categories for a few of the categorical features. If you take the education feature as an example it can be reduced to only 5 or 6 categories if we lump similar levels into 1. Also, the native-country feature had 42 categories, however, we crunched it down to only 2 levels, US and Non-US. That is a whopping 40 columns less than it originally was. Doing such feature transformation on your dataset helps from a common know as curse of dimensionality. As you increase the number of dimentions in your data, it becomes difficult to pull out a signal due to sparsity.

After encoding our categorical variables with one-hot-encoding, it is time to remove them from the predictor variables. We only keep the one-hot-encoded version of them. 

# dropping the original categorical variables

It is time to fit a model on the training dataset and evaluate it with the testing dataset. But the big question is how do we evaluate that our model is doing good? What metric do we use that would give us meaningful result. We should consider choosing the right metric before jumping to fit a model. Common binaray classification metrics include accuracy, f1-score, ROC AUC and PR AUC. We have to analyze the pros and cons of choosing any of them. In this post we will use ROC AUC since our data is skewed and we equally care about both of the target classes.

3.1 XGBoost Parameteres¶

We are going to see a few parameters that we are going to use to build the model.

max_depth[default=6]: refers to the maximum depth of a tree. The higher the value the more complex your model will be.

eta[default=0.3]: learning rate

silent[bool]: refers whether print messages during construction.

eval_metric: refers to evaluation metric. For binary classification in this example we are choosing ROC AUC.

subsample[default = 1]: refers to subsample ratio of the training instances. Setting it to 0.5 means that XGBoost would randomly sample half of the training data prior to growing trees and this will prevent overfitting.

colsample_bytree: refers to the subsample ratio of columns when constructing each tree. Subsampling occurs once for every node constructed.

objective: refers to learning objective. For binary classification, it uses logistic regression to output probability.

seed: refers to random number generator for reproducibility of results

There are many parameters we can add to these, however, those parameters described above are for the sake of example. If you want to check out other parameters, please go to XGBoost official documentation.

dtrain[DMatrix]: data to be trained

num_boost_round[int]: number of boosting iterations

verbose_eval[bool or int]: If True, it prints out the evaluation metric on the validation set at each boosting stage. If verbose_eval = 50, it will print an evaluation metric every 50 boosting stages instead of every boosting stage.

early_stopping_rounds[int]: Activates early stopping. Validation metric needs to improve at least once in every early_stopping_rounds round(s) to continue training.

evals[string]: List of validation sets for which metrics will evaluated during training. Validation metrics will help us track the performance of the model.

maximize[bool]: Referes whether to maximize the eval or not.

3.2 Modeling¶

In this post we are going to use the third party package for XGBoost assuming that we have already installed xgboost in our machine. If you have trouble installing xgboost please refer to the installation guide which will show you how to install xgboost step by step. After we complete installation, we import the xgboost library and create a dictionary for the parameters.

XGBoost uses an internal data structure called DMatrix, which is optimized both for memory efficiency and training speed. We will create two DMatrixs for training and testing datasets. 

# If you haven't installed xgboost, you can install it by typing 
# pip install xgboost
# in your terminal
# Loading the xgboost library
import xgboost as xgb

# Initializing the params
xgb_params = {
    'max_depth':3,
    'eta':0.01,
    'silent':0,
    'eval_metric': 'auc',
    'subsample':0.8,
    'colsample_bytree':0.8,
    'objective':'binary:logistic',
    'seed': 42
}
# Convert the predictor and target variables to Dmatrix data structure 
dtrain = xgb.DMatrix(X_train, y_train, feature_names=X_train.columns.values)
dtest =   xgb.DMatrix(X_test, y_test, feature_names=X_test.columns.values)

# create a list of evaluation criteria for training and test data
evals = [(dtrain, 'train'), (dtest, 'eval')]
xgb_model = xgb.train(params = xgb_params,
                      dtrain = dtrain,
                      num_boost_round = 100,
                      verbose_eval = 25,
                      early_stopping_rounds = 25,
                      evals = evals,
                      maximize = True
                    )

You might be curious to know what the numbers next to each of the leafs mean. The value represnts the score for the class 1 (income >50K). It can be converted into probability using the logistic function. If decisions fell through leaf 7, the probability of class 1 is slightly above 50%. This is the score of a single tree, but if we have 100 boosted trees, we would sum up each leaf and then apply logistic function to get the probability. 

# score from first tree leaf 10
score = 1/(1 + np.exp(-1 * 0.0184805878))
print(f'The probability of class 1 from the first tree at leaf 7 is {round(score,3)}')

The probability of class 1 from the first tree at leaf 10 is 0.505

Let's say the sum of the scores of first 10 trees at leaf 7 is 0.25. Applying logistic function will yield: 

# score from first tree leaf 10
score = 1/(1 + np.exp(-1 * 0.25))
print(f'The score of the first 10 trees at leaf 7 is {round(score,3)}')

The score of the first 10 trees at leaf 7 is 0.562

We can notice that the probability has increased from 0.505 to 0.562. You can now extrapolate that if we have 100 boosted trees the score will increase or decrease, hence, the probability of class 1 decision will be greater or less than 0.5.

In the above feature importance plot, F Score is a metric that simply sums up how many time each feature is split on. You can think of it as frequency score, because it measures the frequency of a given feature is being splitted. This shouldn't be confused with f1 score, a commonly used accuracy score in machine learning for imbalanced dataset.

Capital-gain, education-num, marital-status__Couple, age, capital-loss, hours-per-week and marital-status__Single seem to have high importance in our dataset.

5. Final Thoughts¶

I hope this post In this post we have used XGBoost model to classify people who earn <=50K and >50K on the adult dataset downloaded from the UCI machine learning repository. First we cleaned, transformed and encoded the categorical variables and then we created XGBoost model with 0.89 roc auc score. In this post we created an XGBoost model with only few parameters. We can include other parameters that would increase the roc auc score. For instance, we can tune the scale_pos_weight parameter to account for class imbalance. Or add regularization parameters like $\alpha$ and $\lambda$ represented by reg_alpha and reg_lambda, respectively, which could make our model more robust. Similar to using regularization in the model, we can also also apply kfold cross validation to assess the effectiveness of the model from overfitting the training data.
